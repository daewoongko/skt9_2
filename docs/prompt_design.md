# Prompt Design

## 1. 목적

본 문서는 인지-운동 미션 게임에서 Gemini API를 사용하기 위한 프롬프트 설계 과정을 정리한 것이다.

우리 프로젝트에서는 Gemini를 단순 챗봇이 아니라, 사용자의 운동 결과와 인지 결과를 해석하여 다음 미션을 추천하는 **AI 미션 코치**로 사용한다.

따라서 프롬프트는 다음 조건을 만족해야 한다.

- 사용자 상태를 정확히 반영해야 한다.
- 고령자 대상 안전 조건을 포함해야 한다.
- 인지 과제와 운동 과제를 함께 추천해야 한다.
- 난이도 변화 이유를 설명해야 한다.
- 결과를 서비스에서 사용하기 쉬운 JSON 형식으로 출력해야 한다.

## 2. Baseline Prompt

baseline prompt는 가장 단순한 형태의 프롬프트이다.

주요 특징은 다음과 같다.

- 사용자 데이터를 제공한다.
- 인지와 운동을 함께하는 미션을 추천하라고 요청한다.
- JSON 형식으로 출력하라고 요구한다.
- 세부 안전 규칙이나 난이도 조절 기준은 자세히 주지 않는다.

### Baseline Prompt 예시

```text
아래 사용자 데이터를 보고 인지와 운동을 함께하는 미션을 추천해줘.
반드시 JSON 형식으로 출력해줘.

입력 데이터:
{user_case}

출력 형식:
{
  "mission_name": "...",
  "cognitive_task": "...",
  "movement_task": "...",
  "difficulty": "...",
  "success_condition": "...",
  "reason": "...",
  "safety_note": "..."
}
